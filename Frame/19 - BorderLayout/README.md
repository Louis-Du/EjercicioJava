# BorderLayout

## Tipo
Interfaz Gráfica (Frame/Swing)

## Descripción

Ejercicio que explora el uso del gestor de diseño BorderLayout, uno de los layouts más utilizados en Java Swing. BorderLayout divide el contenedor en cinco regiones: norte, sur, este, oeste y centro.

## Conceptos Practicados

- Gestor de diseño BorderLayout
- Posicionamiento en cinco regiones (NORTH, SOUTH, EAST, WEST, CENTER)
- Comportamiento de redimensionamiento
- Combinación con otros layouts
- Paneles anidados con diferentes layouts
- Diseño de aplicaciones estilo navegador

## Funcionamiento

La aplicación demuestra cómo BorderLayout organiza componentes en las cinco regiones disponibles. La región central se expande para llenar el espacio disponible, mientras que las regiones periféricas mantienen su tamaño preferido.

## Notas

BorderLayout es ideal para diseños tipo aplicación, donde típicamente se coloca una barra de herramientas en el norte, una barra de estado en el sur, paneles laterales en este/oeste, y el contenido principal en el centro.
